# 高中生涯百宝箱Pro - 部署到Gitee和GitHub

本指南将帮助您将项目部署到Gitee和GitHub，包括前端和后端的部署步骤。

## 一、部署前端到Gitee Pages

### 步骤1：创建Gitee仓库
1. 登录 [Gitee](https://gitee.com/)
2. 点击右上角 "+", 选择 "新建仓库"
3. 填写仓库信息：
   - 仓库名称：`gaozhong-baibaoxiang`
   - 仓库介绍：高中生涯百宝箱Pro
   - 选择 "公开" 仓库
   - 不勾选 "使用README文件初始化这个仓库"
   - 点击 "创建"

### 步骤2：上传前端文件
1. 进入新建的仓库
2. 点击 "上传文件" 按钮
3. 选择以下文件和文件夹上传：
   - `index.html`
   - `css/` 文件夹
   - `js/` 文件夹
   - `icons/` 文件夹
   - `manifest.json`
   - `README.md`
   - `DEPLOY.md`
4. 填写提交信息，点击 "提交"

### 步骤3：启用Gitee Pages
1. 进入仓库的 "服务" → "Gitee Pages"
2. 选择 "master" 分支
3. 部署目录选择 "/" (根目录)
4. 点击 "启动"
5. 等待部署完成，复制生成的访问地址

## 二、部署前端到GitHub Pages

### 步骤1：创建GitHub仓库
1. 登录 [GitHub](https://github.com/)
2. 点击右上角 "+", 选择 "New repository"
3. 填写仓库信息：
   - Repository name：`gaozhong-baibaoxiang`
   - Description：高中生涯百宝箱Pro
   - 选择 "Public"
   - 不勾选 "Initialize this repository with a README"
   - 点击 "Create repository"

### 步骤2：上传前端文件
1. 进入新建的仓库
2. 点击 "Add file" → "Upload files"
3. 拖拽或选择以下文件和文件夹：
   - `index.html`
   - `css/` 文件夹
   - `js/` 文件夹
   - `icons/` 文件夹
   - `manifest.json`
   - `README.md`
   - `DEPLOY.md`
4. 填写Commit message，点击 "Commit changes"

### 步骤3：启用GitHub Pages
1. 进入仓库的 "Settings" → "Pages"
2. 在 "Source" 部分，选择 "main" 分支
3. 选择 "/ (root)" 作为目录
4. 点击 "Save"
5. 等待部署完成，复制生成的访问地址

## 三、部署后端服务

由于Gitee和GitHub Pages只支持静态网站，我们需要单独部署后端服务。推荐使用以下免费服务：

### 选项1：Vercel（推荐）
1. 登录 [Vercel](https://vercel.com/)
2. 点击 "New Project"
3. 选择 "Import Git Repository"
4. 导入您的GitHub仓库
5. 配置：
   - Framework Preset：选择 "Other"
   - Build Command：留空
   - Output Directory：留空
   - Environment Variables：添加 `NODE_ENV=production`
6. 点击 "Deploy"
7. 部署完成后，复制生成的API地址

### 选项2：Render
1. 登录 [Render](https://render.com/)
2. 点击 "New" → "Web Service"
3. 连接您的GitHub仓库
4. 配置：
   - Name：`baoxiang-backend`
   - Environment：`Node`
   - Build Command：`npm install`
   - Start Command：`node backend-gitee.js`
   - Plan：选择 "Free"
5. 点击 "Create Web Service"
6. 部署完成后，复制生成的API地址

## 四、配置前端API地址

### 步骤1：修改前端配置
1. 在您的Gitee或GitHub仓库中，编辑 `js/user.js` 文件
2. 找到 `API_BASE_URL` 变量，修改为您的后端服务地址：
   ```javascript
   const API_BASE_URL = 'https://your-backend.vercel.app';
   // 或
   const API_BASE_URL = 'https://baoxiang-backend.onrender.com';
   ```
3. 同样修改 `js/chat.js` 文件中的API地址
4. 提交修改

### 步骤2：更新前端部署
- Gitee Pages和GitHub Pages会自动重新部署
- 等待几分钟后，访问前端地址测试

## 五、测试部署

1. **前端访问**：打开部署后的前端地址
2. **注册/登录**：创建新账号或使用现有账号登录
3. **测试功能**：
   - 点击 "消息" → "小明同学 (AI)" 测试聊天功能
   - 测试添加好友功能
   - 测试健康管理功能
4. **检查API连接**：打开浏览器控制台，查看网络请求是否正常

## 六、常见问题

### 1. 前端显示空白页面
- 检查控制台是否有JavaScript错误
- 确保后端服务正常运行
- 检查API地址配置是否正确

### 2. 聊天功能不工作
- 检查DeepSeek API密钥是否正确
- 检查API密钥是否有额度
- 查看控制台的网络请求

### 3. 好友功能不工作
- 确保后端服务已部署
- 检查API地址配置
- 测试API接口是否可访问

### 4. 部署后访问速度慢
- 考虑使用CDN加速
- 优化图片资源
- 考虑使用更靠近用户的服务器

## 七、项目结构说明

部署时需要包含的文件：
- **前端文件**：index.html, css/, js/, icons/, manifest.json
- **配置文件**：README.md, DEPLOY.md
- **后端文件**：backend-gitee.js, package.json

不需要部署的文件：
- node_modules/
- *.db
- .gitignore
- backend-cloudflare.js（仅用于Cloudflare Workers）
